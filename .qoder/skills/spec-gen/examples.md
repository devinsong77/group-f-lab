## 5. 06 FSD — State 变量补全

**模板**：
```markdown
| 【请补充其余 State】 | | | |
```

**填充后**：
```markdown
| `records` | `QARecord[]` | `[]` | 当前会话的问答记录 |
| `query` | `string` | `''` | 输入框内容 |
| `loading` | `boolean` | `false` | 问答请求是否进行中 |
```

- AC 说"查看历史记录" → 需要 `records`
- AC 说"错误提示" → 需要 `error`
- AC 说"删除确认弹窗" → 需要 `showConfirmDelete`

## 10. 推导原则总结

| 原则 | 说明 |
|------|------|
| **上游优先** | 从上游文档推导，不凭空编造 |
| **AC 驱动** | 05 的每条 AC 都是一个填充线索 |
| **09 字段名统一** | 所有文档引用 09 中冻结的字段名 |
| **标注来源** | 尽量在推导备注中写明推导路径 |
| **教学 vs 生产** | 安全、认证等区分"当前教学版"和"生产建议" |
| **自然演进** | 存储/架构的演进路线要合理，不要跳跃 |
