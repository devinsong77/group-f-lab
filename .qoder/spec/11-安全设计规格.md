## 1. 安全目标与合规要求

> 明确本模块需遵循的安全目标和外部合规标准。投研场景尤其关注数据隐私与内部信息防泄露。

| 安全目标 | 说明 | 对应合规 |
|----------|------|----------|
| 数据不出域 | 所有用户数据、研报内容、问答记录禁止传输至外部服务 | 内部信息安全规范 |
| 最小权限原则 | 每个组件只拥有完成任务所需的最低权限 | 安全设计最佳实践 |
| 审计可追溯 | 所有关键操作形成完整链路日志，支持事后审计与回溯 | 金融行业审计要求 |

## 2. 认证与授权

> 定义用户身份验证方式和权限控制模型。写出认证流程和角色划分。

### 2.1 认证机制

| 项目 | 规格 |
|------|------|
| 认证方式 | 无认证（开放访问）—— 教学版；生产建议对接内部 SSO + JWT Token |
| Token 有效期 | 教学版不适用；生产建议 Access Token 30min，Refresh Token 7d |
| 密码策略 | 教学版不适用；生产建议最少8位，含大小写+数字，90天强制更换 |
| 多因素认证 | 教学版不要求；生产版建议异地登录或敏感操作触发 MFA |

### 2.2 角色与权限矩阵

| 角色 | 研报上传 | 研报阅读 | 问答交互 | 对比分析 | 系统管理 |
|------|----------|----------|----------|----------|----------|
| 分析师 | ✗ | ✓ | ✓ | ✓ | ✗ |
| 管理员 | ✓ | ✓ | ✓ | ✓ | ✓ |
| 研究总监 | ✗ | ✓ | ✓ | ✓ | ✗ |

## 3. 数据安全

> 定义数据分类分级、加密策略、存储安全规范。**投研场景核心关注点**。

### 3.1 数据分类与分级

| 数据类型 | 安全等级 | 示例 | 保护措施 |
|----------|----------|------|----------|
| 研报原文 | **机密** | PDF/HTML 研报文件 | 加密存储，访问审计 |
| 问答记录 | **内部** | 用户提问与AI回答 | 存储加密，定期清理 |
| 用户信息 | **内部** | 用户名、角色、操作日志 | 脱敏展示，最小采集 |
| 系统配置 | 一般 | 模型参数、界面配置 | 版本控制 |

### 3.2 加密策略

| 场景 | 加密方式 | 说明 |
|------|----------|------|
| 传输加密 | TLS 1.2+ | 所有 API 通信强制 HTTPS（教学版可走 HTTP） |
| 存储加密 | 教学版明文 JSON；生产建议 AES-256 | 研报文件与问答记录落盘加密 |
| 密钥管理 | 环境变量（`.env`） | 密钥不硬编码，生产建议使用 KMS 并定期轮转 |

## 4. 接口安全

> 与 `09` API 规格配合，定义 API 层面的安全防护措施。

| 防护措施 | 实现方式 | 说明 |
|----------|----------|------|
| 输入校验 | 服务端白名单校验 | 所有用户输入进行类型、长度、格式校验 |
| SQL 注入防护 | 参数化查询 / ORM | 禁止字符串拼接 SQL |
| XSS 防护 | 输出转义 + CSP | 研报内容渲染时防止脚本注入 |
| CSRF 防护 | SameSite Cookie + 前后端分离 REST API 无状态 | 状态变更接口需 CSRF Token |
| 速率限制 | 【如 100 req/min/user】 | 防止接口滥用 |
| 文件上传校验 | 类型白名单 + 大小限制 | 仅允许 PDF/HTML，限制单文件大小 |

## 5. LLM / AI 安全

> 针对大模型集成的专项安全措施。投研场景数据敏感，需严格管控数据流向。

| 风险点 | 防护措施 | 验收标准 |
|--------|----------|----------|
| 数据外泄至第三方模型 | 本地部署模型 / 内网百炼服务，禁止调用外网 API | 网络出口审计零外传记录 |
| Prompt 注入攻击 | 系统 Prompt 隔离，用户输入预处理过滤 | 注入测试用例 100% 通过，无系统 Prompt 泄露 |
| 模型幻觉 / 错误信息 | 回答标注来源引用，标识 answer_source 与 llm_used | 所有回答均携带来源标签，用户可追溯依据 |
| 敏感信息泄露（模型回复） | 输出过滤层，正则匹配敏感信息并脱敏 | 敏感信息检测测试用例通过率 100% |

## 6. 审计与日志

> 定义安全审计日志的记录范围、保留策略。

| 事件类型 | 记录内容 | 保留期限 | 告警 |
|----------|----------|----------|------|
| 登录 / 登出 | 用户ID、时间、IP、结果 | 180天 | 连续失败 ≥5 次告警 |
| 研报访问 | 用户ID、文档ID、操作类型 | 365天 | 异常批量下载告警 |
| 数据导出 | 导出内容摘要、目标 | 365天 | 每次导出需审批 |
| 系统配置变更 | 变更人、变更前后值 | 永久 | 关键配置变更即时通知 |

## 7. 安全威胁清单（STRIDE 简表）

> 基于 STRIDE 模型，列出主要安全威胁与应对方案。

| 类型 | 威胁 | 风险等级 | 应对措施 | 负责人 |
|------|------|----------|----------|--------|
| S | 伪造用户身份访问研报 | **高** | 强认证 + Token 校验 | 安全团队 |
| T | 研报内容在传输中被篡改 | 中 | TLS + 文件哈希校验 | 安全团队 |
| I | 内部数据通过AI模型泄露 | **高** | 本地部署 + 输出过滤 | 安全团队 |
| D | 大量请求导致服务不可用 | 中 | 速率限制 + 降级策略 | 运维团队 |
| E | 用户操作后否认行为（抵赖） | 中 | 审计日志 + traceId 全链路追踪 | 安全团队 |
| R | 普通用户提权访问管理功能 | 高 | RBAC 角色矩阵 + 接口级权限校验 | 安全团队 |

## 8. 部署安全

> 定义部署架构中的安全要求。

- [ ] 生产/开发环境**严格隔离**，数据不互通
- [ ] 数据库访问仅限应用服务器 IP 白名单
- [ ] 敏感配置（API Key、密码）通过环境变量注入，不入代码仓库
- [ ] 依赖包定期安全扫描（`pip audit` / `npm audit`）
- [ ] 容器镜像定期更新基础镜像，修复已知漏洞
